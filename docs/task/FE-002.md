# FE-002 调整默认语言无前缀路由并保留保留路径命名空间

- **status**: completed
- **priority**: P1
- **createdAt**: 2026-03-09 14:10
- **owner**: codex
- **completedAt**: 2026-03-09
- **relatedPlan**: PLAN-004

## 描述

将当前强制语言前缀的前端路由调整为“默认语言无前缀，非默认语言保留前缀”的结构，并整理保留路径命名空间：

- 默认语言英文使用无前缀路径，例如 `/bkd`
- 中文使用语言前缀路径，例如 `/zh/bkd`
- 系统页面归入 `/sys/*`
- API 继续保留 `/api/*`
- 标签页使用 `/tags/*`
- 分类页使用 `/category/*`
- `link` 作为关系型页面命名空间保留，后续承载 related 语义

## 验收标准

- [x] 明确默认语言与非默认语言的 URL 规则
- [x] 明确 `/sys` `/api` `/tags` `/category` `/link` 的路由边界
- [x] 默认语言页面可直接通过无前缀 URL 访问
- [x] 现有 `/en/*` 路径有明确兼容策略
- [x] 站内链接、语言切换、canonical 与 alternate 规则与新路由一致
